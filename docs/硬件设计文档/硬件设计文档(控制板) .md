## 1. 板卡概述
- 功能定位：控制模拟板的波形生成、与PC进行USB通信、调理输入电源供给模拟板、人机交互。
- 关键性能指标：功耗、尺寸、环境要求
  - 功耗：< 5W（不包括供给模拟板的电源）
  - 尺寸：< 100mm x 100mm
  - 环境要求：0 \~ 40 ℃
- 主要接口：
  - 对外连接器：
    - 电源输入：DC2.1mm 
    - USB 接口：USB Type-B
  - 板间接口：见[板间接口文档](./板间接口设计文档.md)

## 2. 模块详细设计
### 2.1 MCU 最小系统
- **原理图截图**：<font color="red">待补充</font>
  
- **功能描述**：该模块实现什么
  作为整个控制板的核心，负责处理 USB 通信、解析 SCPI 命令、控制 FPGA 配置、管理用户接口（按键/编码器）以及存储用户设置。
- **关键器件**：
  
| 器件 | 型号 | 特性 | 选型理由 |
|------|-----|------|---------|
| MCU | STM32F411CEU6 | ARM Cortex-M4F, 512KB Flash, 128KB RAM | 性能满足需求，成本合理 |
| ESD 芯片 | USBLC6-4SC6Y | 汽车级ESD芯片，低电容 | 保护USB接口 |

- **具体设计**：
  - 电源：
    - VDD: 3.3V，一个单独的10uF电容 + 每一个单独的 VDD 就近放置一个 100 nF 的陶瓷电容, 靠近芯片布局
    - VDDA: 3.3V，就近放置单独的1uF电容 + 100 nF陶瓷电容，靠近芯片布局 <!-- 锁相环是不是用这玩意供电的？ -->
    - VBAT: 3.0V，连接到CR2032电池座，提供实时时钟和备份寄存器电源。
    - VCAP: 连接到一个就近放置的 4.7uF 陶瓷电容，提供内部 LDO 稳压器的稳定电压。
    - VSSA/GND: 通过花焊盘连接到地平面，确保良好的接地
  - 时钟：
    - HSE：外部晶振 8MHz，尽量靠近MCU，内部 PLL 生成 96MHz 系统时钟，匹配电容根据数据手册选型。
    - LSE：外部晶振 32.768kHz，提供 RTC 时钟源，匹配电容根据数据手册选型。
  - 复位：复位引脚连接到调试接口的排针，由调试器控制，并通过一个 100nF 电容连接到地，一个 10kΩ 上拉电阻连接到 VDD（数据手册指出NRST内部已经有了约 40kΩ 的上拉电阻，但噪声环境下不一定可靠，故保留外部上拉电阻焊盘，根据具体情况调整）。
  - 调试接口：7针 2.54mm 排针，合并 SWD 和调试串口(USART1)，用于调试和烧录。
  - Boot 配置：Boot0 默认通过 10kΩ 下拉到地，需要从bootloader模式烧录时，使用按键短接 BOOT0 引脚到 3.3V。
  - SPI1: 连接到板间连接器，MOSI/CLK靠近MCU端串联 22Ω 串阻，MISO不加串联电阻，供 FPGA 配置使用。

- **接口信号**：列出输入/输出信号名称、电平、阻抗、时序要求
  | 信号名 | 方向 | 协议 | 连接到 | 备注 |
  |--------|------|------|--------|------|
  | SWDIO/SWCLK | 双向 | SWD | 调试器（ST-Link） | 用于调试和烧录 |
  | USB DP/DM | 双向 | USB 2.0 FS | USB 接口 | 差分信号，90Ω阻抗 | 
  | SPI1 | 双向 | SPI | FPGA 配置 | MOSI/CLK靠近MCU端串联 22Ω 串阻 |
  | SPI2 | 输出 | SPI | TFT | \ |
  | GPIO | 输入 | GPIO | 按键输入 | 带上拉 |
  |      | 输出 | GPIO | 蜂鸣器 | 三极管扩流 |           
  | IIC | 双向 | IIC | EEPROM | 100kHz |
  | UART | 双向 | UART | 调试串口 | 115200 baud rate |
  
- **保护设计**：
  - SPI1 接口：MOSI/CLK靠近MCU端串联 22Ω 串阻，MISO不加串联电阻，焊盘设计允许焊接 0Ω 电阻或直接短接。
  - USB 接口：见[USB 接口](#USB接口)部分。
  
- **测试点**： 
  - SPI1_SCK/MOSI/MISO/CS：用于示波器观察 SPI 信号质量和时序。
  - IIC_SDA/SCL：用于观察 I2C 通信波形

### 2.2 USB 接口{#USB接口}
 - **原理图截图**: <font color="red">待补充</font>
- **功能描述**：
  - 通过 USB-FS 接收 SCPI 命令
  - 通过 USB-FS 发送响应数据
  - 自动检测线缆插入
- **关键器件**：

| 器件 | 型号 | 特性 | 选型理由 |
|------|-----|------|---------|
| USB 接口 | USB Type-B | 4线母头 USB2.0 | 连接稳定，可靠性高，行业标准 |
| ESD 芯片 | PRTR5V0U2X | 超低电容双轨对轨ESD芯片 | 保护USB接口 |

- **接口信号**：
  | 信号名 | 方向 | 协议 | 连接到 | 备注 |
  |--------|------|------|--------|------|
  | USB_DP/DM | 双向 | USB 2.0 FS | PC | CDC |
  | VBUS | 输入 | 电平 | MCU GPIO | USB 连接检测 |

- **具体设计**：
  - USB DP/DM 走差分线，阻抗控制在 90Ω，长度匹配，避免放置过孔, 靠近接口处放置 ESD 保护芯片。
  - USB 串联 22Ω 串阻可减少反射和 EMI，但ST官方不建议添加，保留焊盘，靠近 MCU 端，但焊接0Ω电阻或直接短接。
  - GND 与板卡地平面直接连接。
  - 连接器外壳通过花焊盘与地连接，提供额外的屏蔽和接地路径。
  - VBUS 线通过分压电阻分压到3.3V后连接到 MCU 的 GPIO，用于检测 USB 连接状态。
  - DP 上拉由 MCU 内部实现，无需外部 1.5k 电阻，软件配置即可。


- **保护设计**：ESD、过流、过压、防反接等
  - ESD 保护：USBLC6-4SC6Y 汽车级芯片，低电容，保护 USB DP/DM 线免受静电放电损伤。
  - 过流保护：USB 供电线（VBUS）只做检测，不直接供电，避免过流风险。
  - 防反接：USB 接口本身设计为防反插，内部电路不需要额外防反接措施。
  
- **测试点**：
  - TP_USB_DP/DM：用于示波器观察 USB 信号质量和时序。
  - TP_USB_VBUS：用于测量 USB 连接状态电压，验证连接检测功能。

### 2.4 人机接口（v3.0实现）
> v1.0、v2.0 只支持SCPI 远程控制，v3.0支持本地操作
- **原理图截图**: <font color="red">待补充</font>
- **功能描述**：进行人机交互
- **关键器件**：

| 器件 | 型号 | 特性 | 选型理由 |
|------|-----|------|----------| 
| 显示屏 | 3.5寸 TFT | MIPI接口，1024x768 分辨率 | 驱动方便 |
| 编码器 | EC11-01 | 360度旋转，带按键 | 可以简化面板 |

- **具体设计**：
  - 屏幕接口通过 MIPI 输出，差分对走线，长度匹配，避免放置过孔。
  <!-- MIPI走线细节待补充 -->
  - 编码器旋转通过定时器编码器接口检测，按键通过 GPIO 中断检测，内部上拉。  
  - 按键排列成矩阵键盘，通过行列扫描法扫描键值。
  
- **接口信号**：
  
| 信号名 | 方向 | 协议 | 连接到 | 备注 |
|--------|------|------|--------|------|
| TFT_MIPI | 输出 | MIPI | TFT | \ |
| ENCODER_A/B/C | 输入 | 按键/编码器 | 编码器 | \ |
| 按键 | 输入 | 按键 | 按键输入 | 矩阵键盘 |
- **测试点**：
  - TP_TFT_MIPI：用于示波器观察 TFT 屏信号质量。
  - TP_ENCODER_A/B/C：用于示波器观察 编码器/按键信号质量。

  | 信号名 | 方向 | 协议 | 连接到 | 备注 |

### 2.6 电源输入与保护
- **原理图截图**（或链接）
- **功能描述**：  
  - 将输入的 24V 转为 ±17V_M、3.3V_D 电压轨，为控制板和模拟板提供稳定电源
  - 过流、过压、反接保护

- **关键器件**：

  | 器件 | 型号 | 特性 | 选型理由 |
  |------|-----|------|----------|
  | 电源输入接口 | DC2.1mm | 2.1mm DC 电源插座 | 行业标准，连接稳定，从根源上避免反接 |
  | TVS 二极管 | SMBJ24A | 24V 钳位电压，单向导通，SMB 封装 | 保护电源输入免受过压冲击 |
  | +17V_M DC-DC | TPS54360 | 3A 输出电流，24V 输入，效率高 | 用于主电源轨，效率更高，降低热量 |
  | -17V_M DC-DC | TPS54360_反向buck | 3A 输出电流，24V 输入，效率高 |  |
  | +3.3V_D DC-DC | TPS54360 | 3A 输出电流，24V 输入 | 用于数字电源轨，效率更高，降低热量 |

2. **注意事项**
  - DCDC 电路的低电压端（正向 buck 的 GND，反向 buck 的 Vout-）都通过大量过孔连接到对应的铜皮以保证回流路径低阻抗
  - SW 节点面积尽量小，且远离敏感信号线
  - 热焊盘通过大量过孔接到底层铜箔（GND 或 Vout-）
  - 输出电容可适量加大，选择大容量的电解电容配合低 ESR 的 MLCC ，可以确保稳定性和降低纹波
  - 输入端加大电容，降低输入纹波对电路的影响，需要注意电容的耐压值

## 3. PCB 布局与布线指南
- 层叠结构（例如 4 层：信号-地-电源-信号）
- 关键信号的处理：
  - 高速时钟（差分/单端）的包地或阻抗控制

- 散热设计（热过孔、散热铜皮、散热器安装）

## 4. 测试与调试
### 4.1 测试点位置
所有电源轨、关键信号线（如 SPI 总线、USB DP/DM）预留测试点
### 4.2 上电前检查清单（短路测试、电源电压测量）
### 4.3 单板独立测试步骤：
分步上电：先焊电源部分，验证各电压轨；再焊STM32，烧录代码；接着焊外设验证USB，EEPROM等是否正常，最后焊连接器，测试与模拟板的通信。
关键波形测试点：
  - USB DP/DM 波形（示波器观察）
  - SPI 信号（时钟、数据）波形
模块测试：
  - 电源测试（空载、负载纹波）
  - 数字通信环回测试（SPI、I2C）
  - 模拟通道静态测试（DAC 输出固定码、测量电压）
  - 动态测试（用示波器观察波形）
- 常见故障排查指南
